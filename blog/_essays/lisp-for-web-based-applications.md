## 요약 (Summary)

🎯 웹 애플리케이션에 Lisp을 쓰면 어떨까요?

✨ 핵심 내용 요약

Paul Graham이 Viaweb을 만들 때 Lisp을 선택한 이유: **웹 애플리케이션에 완벽했기 때문입니다**.

웹 앱의 특징: (1) 계속 변합니다 - 매일 업데이트, (2) 인터랙티브합니다 - 사용자 입력에 반응, (3) 복잡합니다 - 데이터베이스, 사용자 세션, 복잡한 로직. Lisp의 장점: (1) **빠른 개발** - 적은 코드로 많은 것을 만듭니다, (2) **즉시 테스트** - 코드를 바로 실행해서 테스트합니다, (3) **쉬운 유지보수** - 코드가 간결하니까 수정도 쉽습니다. Graham의 경험: 경쟁자가 C++로 1개월 걸린 기능을 Lisp으로 1주일에 만들었습니다. 이게 Viaweb이 이긴 이유입니다. 스타트업 교훈: **빠른 개발이 경쟁 우위**입니다.

**핵심 포인트**
• 웹 앱은 빠른 개발과 변경이 중요합니다
• Lisp은 웹 앱에 최적화되어 있습니다
• 빠른 개발 속도가 경쟁 우위입니다

🚀 오늘 바로 실천해볼 한 가지
"어떤 도구가 가장 빠르게 개발할 수 있게 하나?"를 기준으로 기술 스택을 선택하세요.
